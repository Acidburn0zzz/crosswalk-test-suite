Test Steps:

  1. Open ADT by a menu entry to start it, or run the 'eclipse/eclipse' command from inside your Android SDK's root directory
  2. In the File menu, choose Import....
  3. In the Import dialog box, choose Android then Existing Android Code into Workspace. Click the Next button
  4. Choose the directory for the generated project; for example, for the HelloWorld project
  5. Click the Finish button to import the projects
  6. Check the error message of the import project
    • If the error massage display 'The import org.apache.cordova cannot be resolved'
    • If the error massage display 'The import com.squareup.okhttp.OkHttpClient cannot be resolved'
  7. Select the project, then Run as > Android Application and check the console message
  8. Check the test device
  9. You may need to turn off the automatic build option (uncheck Project > Build Automatically in Eclipse) so you can build the projects manually and in the correct order

Expected Result:

  1. Open the eclipse
  2. Open the menu
  3. Open file browser window
  4. Select the project
  5. The project can be import successfully
  6. Analysis the error message and resolve
    • Execute command: cd /localpath/crosswalk-cordova-xxx-arm/framework/src then copy the org file to the import project
    • Copy the com file to the import project
  7. The project can be build and the console display 'Installing *.apk Success!'
  8. The project can be launch successfully
  9. The project can be build and launch successfully
